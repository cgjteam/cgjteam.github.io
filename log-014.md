# Log 014 — What Is Actually Being Formalized?

### Introduction
At first sight, the answer seems obvious: formal verification proves the theorem. However, closer inspection shows that this is only partially true. The theorem is the final statement, but the actual object being formalized is the proof itself. Understanding this distinction is essential for the methodology developed in this project.

### The Theorem vs. The Proof
* **The Theorem** states that the three medians of a triangle are concurrent. This mathematical statement is independent of any particular proof; it can be established synthetically, analytically, using vectors, or barycentric coordinates. The theorem remains the same across all these approaches.
* **The Proof** is different. It specifies which constructions are introduced, which previous theorems are used, how intermediate results are connected, and in what order the logical dependencies appear. Changing the proof changes this structure without changing the theorem.

### Finlay's Construction
Ian Finlay's contribution is not a new theorem, but a particular proof architecture. The purpose of CGJteam Lab is therefore not merely to formalize the concurrency theorem, but to formalize Finlay's construction itself. This distinction explains why the logical analysis performed in the previous logs was a necessary prerequisite.

### Consequences for Formal Verification
A formal proof assistant verifies a specific chain of reasoning. Different classical proofs of the same theorem will generally produce different formal developments. Consequently, the primary object of this project is the architecture of Finlay's proof rather than the theorem in isolation. 

By formalizing the *construction*, we create a set of reusable logical components that are not bound solely to the concurrency of medians.
