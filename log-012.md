# Log-012: Eliminating the First Axiom

**Date:** 2026-07-18  
**Project:** CGJteam Lab  
**Status:** Library Refactoring

---

## 1. Objective

Previous logs established a complete formal proof of the Midsegment Theorem.

At that stage, however, the geometry library still contained the declaration

```lean
axiom MidsegmentTheorem
```

Although mathematically justified by the newly developed proof, the theorem remained an axiom from the perspective of Lean.

The objective of the present work was to eliminate this axiom while preserving the existing public interface of the library.

---

## 2. A More Natural Theorem

The proof developed during the formalization naturally led to a theorem with a different interface.

Instead of reproducing the original statement directly, the theorem was formulated as

```lean
theorem MidsegmentParallel
```

whose hypotheses describe two midpoints sharing a common endpoint.

This formulation follows the geometry of the proof itself and proved considerably more natural for formal reasoning.

The theorem was proved independently of the former axiom.

---

## 3. A Small but Important Difference

Replacing the axiom was not immediate.

Although the original theorem and the new theorem are mathematically equivalent, they use different orientations of the midpoint relation.

This distinction is usually ignored in traditional mathematical writing, but Lean treats the predicates

```text
IsMidpoint M A B
```

and

```text
IsMidpoint M B A
```

as different propositions.

To bridge this gap, an auxiliary theorem was introduced:

```lean
theorem MidpointSymmetry
```

This theorem establishes that the midpoint relation is invariant under exchanging the endpoints of a segment.

Its proof relies only on already available properties of collinearity and segment congruence.

---

## 4. Compatibility with the Existing Library

Using the symmetry theorem, a compatibility wrapper was constructed.

The wrapper has exactly the same interface as the former axiom:

```lean
theorem MidsegmentTheorem
```

Internally, however, it is now derived entirely from the formally verified theorem

```lean
MidsegmentParallel.
```

This design preserves backward compatibility with every existing proof in the project.

---

## 5. Removing the Axiom

After introducing the compatibility wrapper,

```lean
axiom MidsegmentTheorem
```

was removed from `GeometryBase`.

The complete project, including `FinlayProof`, compiled successfully without any modifications to the mathematical proof.

This confirms that the former axiom has been completely replaced by a verified theorem.

---

## 6. Significance

This is the first successful elimination of an axiom in the CGJteam Lab geometry library.

The importance of this result extends beyond a single theorem.

It demonstrates that an axiomatic component of the library can be replaced by a formal proof while preserving the existing API and without requiring changes in dependent developments.

The adopted workflow is therefore expected to become the standard procedure for future refactoring:

1. Prove a mathematically natural theorem.
2. Introduce only the minimal auxiliary lemmas required.
3. Build a compatibility wrapper preserving the original interface.
4. Remove the corresponding axiom.
5. Verify the entire library by recompilation.

This methodology allows the geometry library to evolve incrementally from an axiomatic framework toward a completely verified formal development while maintaining stability for all existing formal proofs.

---

## 7. Outlook

The successful elimination of `MidsegmentTheorem` marks the beginning of a broader refactoring effort.

Future work will follow the same methodology to replace additional axioms by formally verified theorems.

Each completed step will reduce the axiomatic core of the library and increase the proportion of results established entirely within Lean.

The long-term objective is to obtain a geometry library in which the smallest possible collection of primitive assumptions supports an extensive hierarchy of formally verified synthetic geometry.
