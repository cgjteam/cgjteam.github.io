# Log 011 - Documenting the Geometry Core

## Why Document the Axioms?

As the first formalization matured, an interesting observation emerged.

The Lean declarations precisely specify the logical interface of the geometry library, but they are not always the most convenient way to understand the underlying mathematics.

Every primitive axiom should therefore be accompanied by a short mathematical description referring directly to its arguments.

## Proposed Documentation Table

| Lean declaration | Mathematical meaning | Typical conclusion |
|------------------|----------------------|--------------------|
| `midpoint_collinear A M B` | `M` is the midpoint of `AB`. | `A, M, B` are collinear. |
| `collinear_swap A B C` | The order of points on a line may be changed. | `A,B,C -> A,C,B`. |
| `MidsegmentTheorem A B C D E` | `D` and `E` are the midpoints of `AC` and `BC`. | `DE || AB`. |
| `ParallelCollinearLeft A B C D E` | Move one endpoint of a parallel line along the same supporting line. | `AB || CD -> EB || CD`. |
| `ParallelSymm A B C D` | Parallelism is symmetric. | `AB || CD -> CD || AB`. |
| `ParallelFlipRight A B C D` | Reversing one line preserves parallelism. | `AB || CD -> AB || DC`. |
| `ParallelogramOfParallel A B C D` | Two pairs of opposite parallel sides determine a parallelogram. | `AB || CD`, `BC || AD` -> `ABCD` is a parallelogram. |
| `ParallelogramDiagonals A B C D M` | `M` is the intersection of the diagonals. | `AM = MC`, `BM = MD`. |
| `MidpointMedian A B C M` | `M` is the midpoint of `BC`. | `AM` is a median of triangle `ABC`. |
| `IntersectionOnSameLine ...` | Transfer an intersection to another representation of the same line. | See Lean declaration. |
| `CollinearTrans ...` | Propagate collinearity through a common supporting line. | See Lean declaration. |

## Why This Matters

This table documents the mathematical API rather than the Lean implementation.

It helps readers understand the geometry core without reading every declaration, while also serving as a concise reference during further formalization.

As the library grows, an additional **Used in** column can record where each axiom first appears (for example, `FinlayStep1` or `FinlayStep4`).

## Conclusion

A compact axiom table bridges the gap between formal code and mathematical intuition. It is an effective form of documentation for a growing geometry library.
