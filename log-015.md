# Log 015 — Designing the Formal Geometry Library

## Introduction

The previous logs focused on understanding Ian Finlay's synthetic proof and reconstructing its logical architecture. Having identified the essential definitions, axioms, intermediate lemmas, and their dependencies, the project now reaches a new stage. The question is no longer how the proof works, but how the mathematical knowledge required by the proof should be organized for formal verification.

The objective of this log is therefore not to discuss another geometric theorem, but to introduce the architecture of the formal geometry library that will support the entire Lean development.

## Why a Geometry Library?

Formal verification is rarely about proving a single isolated theorem. Instead, it requires a coherent mathematical environment in which definitions, constructions, and previously established results can be reused.

Rather than creating a separate Lean file for every theorem, CGJteam Lab adopts the opposite philosophy: a single evolving geometry library containing reusable mathematical components. The concurrency theorem for triangle medians is simply the first theorem developed within this library.

## The Architecture

The current organization of the project follows four conceptual layers:

1. **Definitions** — the fundamental geometric vocabulary (points, collinearity, congruence, midpoints, parallelism, etc.).
2. **Axioms** — the primitive geometric principles accepted without proof.
3. **Derived Lemmas** — reusable intermediate results constructed from the axioms.
4. **Main Theorems** — complete formal proofs built upon the previous layers.

This hierarchy reflects logical dependencies rather than the chronological order in which the code is written.

## Documentation and Implementation

The project deliberately separates documentation from implementation.

The mathematical architecture of the library is maintained in the [Geometry Library Index](./Geometry.md), which records identifiers, dependencies, implementation status, and relationships between formal objects.

The formal implementation itself is contained in the single Lean source file [Geometry implementation](./Geometry_lean.html). This separation allows the documentation to remain stable even as the implementation evolves through refactoring and optimization.

## Research Perspective

The goal of the project is no longer the formalization of a single proof. Instead, the objective is to build a reusable synthetic geometry library from which many classical geometric proofs may eventually be developed. Finlay's proof serves as the initial case study for designing this architecture, but it is not the final destination of the project.
