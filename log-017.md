# Log-017
# Suppes as an Alternative Midsegment Route

**Date:** 2026-07-22
**Project:** CGJteam Lab
**Status:** Alternative Axiomatic Development

---

## 1. Overview

The reconstruction of Suppes' Theorem 11 has now been completed in Lean.
The theorem constructs a primitive parallelogram from the three midpoints
of a triangle and provides a constructive route to the Midsegment Theorem.

This development is not intended to replace the existing synthetic proof
in `MidsegmentParallel.lean`. Instead, it provides an alternative route
whose starting point is Suppes' quantifier-free affine geometry.

---

## 2. Two Parallel Developments

The existing library contains a reusable proof of the Midsegment Theorem
based on constructions, congruence, parallelograms and collinearity.

The new module

```lean
MidsegmentParallelSuppes.lean
```

derives the same geometric conclusion from Suppes' midpoint, doubling and
collinearity operations. The original modules remain unchanged.

The project can therefore compare two formal paths to the same theorem:

```text
synthetic construction  -> MidsegmentParallel
Suppes affine axioms    -> MidsegmentParallelSuppes
```

---

## 3. Connecting the Two Languages

Suppes' formal language does not take parallelism as primitive. It first
constructs primitive parallelograms and then defines parallel segments.

The geometry library, by contrast, expresses the result through
`Geo.Parallel`. The class `SuppesMidsegmentBridge` makes this interface
explicit: it records the interpretation of midpoint, collinearity and
Suppes parallelism in the shared `GeometryBase` language.

The degenerate case is also stated explicitly, since Suppes' definition of
parallel segments excludes collinear configurations.

---

## 4. Finlay's Proof

A new file

```lean
FinlayProofSuppes.lean
```

repeats the structure of Finlay's proof while importing the Suppes-based
midsegment module. The original `FinlayProof.lean` remains the stable
reference version.

This preserves the existing development and creates a controlled setting
for comparing alternative axiomatic foundations of the same proof.

---

## 5. Current Status

Both new modules compile successfully. The Suppes axioms and theorems were
placed in the namespace `Geometry.Suppes`, allowing them to coexist with
the existing `GeometryBase` vocabulary without name collisions.

The next step is to document the bridge assumptions and the relation
between the two midsegment developments in the project wiki.
