# Log 007 — The Geometry Core: Definitions Before Theorems

## Introduction

The previous logs described the transition from a single formal proof toward a reusable geometry library.

After completing the formalization of Finlay's proof, however, a more fundamental question appeared:

What is the actual mathematical core required by the proof?

The original geometric argument is usually presented using a rich intuitive language. A mathematician naturally thinks about points, lines, segments, lengths, angles, and constructions.

The formal development revealed a different picture.

A large part of the classical geometric language was not required by the proof.

The purpose of this log is to examine the minimal geometric structure that emerged from the formalization.

---

## The Geo Structure

The central object of the current library is the abstract structure:

```lean
Geo
```

It does not represent Euclidean geometry directly.

Instead, it provides only the primitive objects and relations required by the formal developments.

The current structure contains:

- points,
- collinearity,
- parallelism,
- betweenness,
- congruence,
- midpoint,
- median,
- parallelogram.

The important observation is that these notions are not implemented through coordinates or metric formulas.

The library does not begin with:

- a coordinate plane,
- distance functions,
- angles,
- vectors,
- transformations.

Instead, it starts from geometric relations.

This reflects the synthetic nature of the project.

---

## Geometry Without Coordinates

A traditional analytic approach would represent points by numerical coordinates and define geometric concepts through equations.

Such an approach is powerful, but it introduces a large amount of additional structure.

The Finlay formalization showed that none of this was necessary.

The proof uses only statements such as:

- three points are collinear,
- two segments are parallel,
- a point is a midpoint,
- diagonals of a parallelogram bisect each other.

The formal proof therefore operates at a higher conceptual level.

The geometry is described through relations rather than through calculations.

---

## The Midpoint Investigation

The midpoint concept provides one of the clearest examples.

In informal geometry, a midpoint usually carries several meanings simultaneously:

- equal distances,
- belonging to a segment,
- symmetric position,
- division of a segment into two equal parts.

During formalization, these interpretations separated.

The Finlay proof required only the consequences expressed by the library interface.

For example:

```lean
Geo.IsMidpoint M A B
```

was used to obtain collinearity and to apply the Midsegment Theorem.

No explicit metric computation was required.

This illustrates one of the main advantages of formalization:

it separates intuitive meaning from logically necessary assumptions.

---

## Missing Concepts Are Also Information

An important result of the formalization is not only what was included, but also what was absent.

The proof of Finlay did not require:

- lengths,
- angle measurements,
- triangle similarity,
- coordinate systems,
- ordering of points on a line,
- circle constructions.

These concepts are natural in classical geometry, but they are not part of the logical dependency chain of this theorem.

Their absence is mathematically meaningful.

It identifies the actual structural level at which the theorem exists.

---

## Synthetic Geometry as a Layered Theory

The current development suggests a layered architecture.

At the lowest level:

```
Points

Relations

Basic geometric predicates
```

Above this:

```
Midpoints

Midsegment constructions

Parallelism
```

Then:

```
Parallelogram theory

Diagonal properties

Median properties
```

Finally:

```
Classical synthetic theorems
```

Each layer depends only on the previous one.

This structure is different from a textbook presentation, where many geometric ideas are introduced simultaneously.

Formalization forces these dependencies to become explicit.

---

## Consequences for Library Design

The first formal proof suggested that a geometry library should not be organized only as a collection of theorems.

It should also expose the conceptual layers behind those theorems.

The question is not only:

> What theorems can be proved?

but also:

> Which primitive ideas generate the largest mathematical structures?

For the current project, midpoint, parallelism and parallelograms appear to be such generating concepts.

---

## Outlook

The next stage of development will investigate how these primitive concepts generate larger theories.

The goal is not to reproduce all of Euclidean geometry immediately.

Instead, the project will continue by identifying reusable mathematical structures that naturally emerge from formal proofs.

The Finlay theorem provided the first experiment.

The geometry core extracted from it provides the foundation for future developments.
