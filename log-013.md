# Research Log 013

# Refactoring the Incidence Layer

## Motivation

The original version of the Geometry library treated collinearity as a primitive relation stored directly inside the geometry object.

```lean
structure Geo where
  Point : Type u
  Collinear : Point -> Point -> Point -> Prop
```

Although convenient, this design mixed together two different logical levels.

At the primitive level, geometry consists of objects such as points and lines together with their incidence relation.

At a higher level, collinearity is not primitive. It is a derived concept expressing that three points belong to the same line.

The goal of this refactoring was therefore to separate these two layers.

---

## Simplifying the Geometry Core

The geometry kernel was reduced to its primitive objects.

```lean
structure Geo where
  Point : Type u
  Line  : Type u
```

The primitive incidence relation was moved into a dedicated typeclass.

```lean
class HilbertIncidence where
  OnLine :
    Geo.Point ->
    Geo.Line ->
    Prop
```

This change makes the logical foundation closer to Hilbert's incidence axioms while keeping the remainder of the library independent of any particular implementation.

---

## Defining Collinearity

Instead of treating collinearity as primitive, it is now defined by incidence.

```lean
def PrimCollinear
    [HilbertIncidence Geo]
    (A B C : Geo.Point) : Prop :=
  ∃ l : Geo.Line,
    OnLine A l ∧
    OnLine B l ∧
    OnLine C l
```

The public interface continues to use

```lean
Collinear Geo A B C
```

which is now simply an abbreviation for the incidence-based definition.

This preserves backward compatibility while changing the logical implementation underneath.

---

## Eliminating Derived Axioms

Several elementary properties of collinearity no longer need to be postulated.

For example,

- cyclic permutation,
- symmetry,
- rotation,
- extraction of the witnessing line,

are now ordinary Lean theorems obtained directly from the existential definition.

These results replace former axioms by formal proofs.

---

## Migrating the Library

The refactoring propagated through the complete dependency chain.

```
GeometryCore
        ↓
HilbertAxioms
        ↓
GeometryBase
        ↓
MidsegmentParallel
        ↓
FinlayProof
```

Each module was migrated while preserving its mathematical content.

The existing formal proof of Finlay's construction required interface adjustments but no conceptual modifications.

---

## Result

After completing the migration,

- the primitive predicate `Geo.Collinear` disappeared,
- collinearity became a derived notion,
- the incidence layer became an independent foundation,
- the complete Finlay development compiled successfully on the new architecture.

No mathematical theorem had to be changed.

Only the logical organization of the library was improved.

---

## Significance

This is the first architectural refactoring of the project that changes the logical kernel without changing the mathematical developments built on top of it.

The experiment demonstrates that the Geometry library can be systematically reduced to a smaller collection of primitive notions while preserving all existing formal proofs.

This establishes a methodology for future work: replacing temporary axioms with formally verified theorems while maintaining a stable public interface.

The long-term objective remains the same: to identify the minimal logical assumptions required by classical synthetic geometry.
